# CSS 규칙 충돌?
CSS의 역할은 한 Markup Element, HTML에서는 HTML Element에 대해 꾸며주는 역할을 합니다. CSS 규칙 충돌은 한 HTML Element에 대해 여러가지 규칙이 적용될 수 있는 상황을 이야기 합니다.

우리는 논의 중 충돌이 발생하면 합의된 기준을 두고 점수를 매겨 우선순위를 산출합니다. 기준을 합의할 때, 각 기준은 명확하고 측정이 가능해야 합니다. CSS에서는 어떻게 이런 기준을 정하고, 어떻게 적용하고 있을까요? 이 것을 알아서 어떤게 좋을까요?

이 글을 통해 우리는 CSS의 기본 컨셉인 Inheritance, Specificity, Cascading에 대해 알아볼 것입니다. 그를 통해 특정 상황에서 어떤 규칙이 적용될 것인지 인지하고, CSS 자체가 아닌 우리가 당면한 진짜 문제를 해결하는데 시간을 쏟을 수 있게 됩니다. 이 글은 [다음 글](https://developer.mozilla.org/en-US/docs/Learn/CSS/Building_blocks/Cascade_and_inheritance)을 많이 참조하였으며, 시간날 때 보시는 것을 추천드립니다.
